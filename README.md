# EIA Data Explorer — Python + Chart.js

Экзаменационный проект: визуализация данных открытого **EIA API v2**
(U.S. Energy Information Administration, https://www.eia.gov/opendata/)
через собственный GUI с фильтрами и выбором периода 1970–2025.

## Архитектура (без микросервисов, один процесс)

```
Браузер (index.html + Chart.js)
        │  fetch('/api/...')
        ▼
Flask (app.py)  ──proxy──▶  https://api.eia.gov/v2/...
```

Flask ничего не знает заранее про конкретный под-API (coal / petroleum /
natural-gas / electricity / total-energy / co2-emissions / seds / ...).
Он просто прокидывает запросы к EIA и отдаёт JSON как есть. Вся структура
(доступные facet'ы, частота, колонки данных) читается **на лету** из
метаданных, которые сам EIA отдаёт для любого маршрута — поэтому фронт
одинаково работает с любым под-API из каталога, достаточно кликами
дойти до нужного узла (обычно `.../data`).

## Установка

```bash
pip install -r requirements.txt
```

## Ключ API

1. Зарегистрироваться (бесплатно, мгновенно): https://www.eia.gov/opendata/register.php
2. Создать файл `.env` рядом с `app.py`:
   ```
   EIA_API_KEY=ваш_ключ_сюда
   ```

## Запуск

```bash
python app.py
```

Открыть http://127.0.0.1:5000

## Как пользоваться GUI

1. **Шаг 1** — кликами по «чипам» спускаетесь по дереву каталога EIA
   (например: `coal` → `data`, либо `petroleum` → `data`, либо
   `electricity` → `retail-sales`). Хлебные крошки сверху позволяют
   вернуться назад.
2. Как только маршрут становится «листовым» (в нём есть facets/frequency/data),
   автоматически открывается **Шаг 2 — Фильтры**:
   - частота (annual / monthly / ...),
   - какие числовые показатели тянуть (value, production, stocks и т.п.),
   - фильтры по каждому facet'у (тип угля, штат, сектор и т.д.) — значения
     подгружаются с сервера,
   - диапазон лет 1970–2025 двойным ползунком.
3. Кнопка **«Построить график»** дергает `/api/data/<route>` и рисует
   график через Chart.js (линия/столбцы), группируя серии по выбранным
   facet-фильтрам.

## Куда развивать (следующий этап — React Native)

Бэкенд (`app.py`) уже полностью независим от фронта — это чистый REST-прокси
(`/api/routes`, `/api/facet`, `/api/data`). Поэтому на следующем этапе,
когда нужно будет подключить React Native, достаточно переиспользовать те же
3 эндпоинта и просто заменить `static/*` на экраны React Native — сам Flask
трогать не придётся.
